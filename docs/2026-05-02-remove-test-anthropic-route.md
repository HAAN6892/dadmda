# 임시 connection test 라우트 삭제

> **작업 범위**: `src/app/api/test/anthropic/route.ts` 한 파일 삭제 + 관련 흔적 정리
> **예상 시간**: 5~10분
> **선행 컨텍스트**: `.claude/memory/session_handoff.md` 또는 가장 최근 핸드오프 참고

---

## 1. 작업 목적

Anthropic SDK 인프라 검증(connection test)이 끝났다. 임시 라우트를 그대로 두면:

- 인증 가드 없는 API 엔드포인트가 배포 시 그대로 노출될 위험
- 페르소나 추출 API Route 작업 시 "임시 코드 잔존" 형태의 스코프 오염 발생
- 후속 세션에서 이 라우트를 참고해서 잘못된 패턴 재생산 가능성

따라서 페르소나 추출 본 작업 들어가기 직전인 지금 정리한다.

---

## 2. 작업 단계 (단계별 보고 + 컨펌)

### 단계 1: 현재 상태 확인

다음 항목을 보고하고 사용자 컨펌 대기:

1. `src/app/api/test/anthropic/route.ts` 파일 존재 확인
2. 이 라우트를 import/참조하는 다른 파일이 있는지 확인
   - 명령: `grep -r "api/test/anthropic" src/` (또는 동등한 검색)
3. 위 결과를 사용자에게 보고

**중단 조건**: 만약 참조하는 파일이 있으면 보고 후 사용자 지시 대기. 임의로 같이 수정하지 말 것.

### 단계 2: 파일 삭제

단계 1 컨펌 받은 후:

1. `src/app/api/test/anthropic/route.ts` 삭제
2. 빈 디렉터리 정리:
   - `src/app/api/test/anthropic/` 빈 폴더 삭제
   - `src/app/api/test/` 폴더가 비었으면 그것도 삭제
3. 삭제 결과 보고 (어떤 경로가 사라졌는지 명시)

### 단계 3: 검증

1. `npm run build` 실행 (또는 사용자 환경 기준 빌드 명령)
2. 타입 에러 / 빌드 에러 없는지 확인
3. 결과 보고

**실패 시**: 에러 전문을 그대로 보고하고 사용자 지시 대기. 임의 수정 금지.

### 단계 4: 커밋 메시지 제안

다음 형식으로 커밋 메시지 초안 제안 (실제 커밋은 사용자 컨펌 후):

```
chore: connection test 임시 라우트 제거

- src/app/api/test/anthropic/route.ts 삭제
- Anthropic SDK 인프라 검증 완료에 따른 정리
- 페르소나 추출 API Route 작업 전 임시 코드 청소
```

### 단계 5: 커밋 (사용자 컨펌 후)

1. `git add` (해당 파일/폴더만)
2. `git commit` (위 메시지로)
3. **`git push` 절대 실행 금지** (사용자가 직접)
4. 커밋 해시 보고
