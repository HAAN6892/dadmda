# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 서비스 개요

**dadmda (다듬다)** — "개떡같이 던지면 찰떡같이 다듬어줍니다"

한국 초등교사가 학부모 메신저(하이톡·카톡·문자) 대화를 복붙 임포트하면, AI가 학부모별 페르소나(관계의 결)를 자동 추출하고 비식별화한다. 새 메시지 작성 시 4축(격식·공감·단정·길이) 슬라이더로 미세 조정해 페르소나·이전 맥락을 반영한 메시지를 다듬어주는 웹 도구.

### 타깃 시장
- **1차**: 한국 초등교사 (배우자가 첫 사용자, dogfooding)
- **확장**: 직장인(동료·상급자), 외부 기관(교육청 등), 의료진
- 데이터 모델은 도메인 무관하게 설계 — `relationship_type` 필드로 구분, UI 라벨·AI 프롬프트는 변수화

## 핵심 원칙 (작업 판단 기준)

1. **확장 가능한 구조** — 도메인 무관 데이터 모델. 학부모 전용 필드를 만들지 말 것. 공통 추상화 + `relationship_type` 분기.
2. **핵심 흐름 완성도 우선** — 부가 기능보다 1주차 4개 화면(로그인/홈/페르소나 만들기/메시지 작성)의 완성도를 우선.
3. **사용자 귀찮음 0** — 강제 입력·튜토리얼·확인 팝업 최소화. 디폴트로 통과시키고, 필요한 사람만 손대게.
4. **자동 비식별화는 임포트 텍스트 한정** — 사용자가 직접 입력한 값(페르소나 이름, 메모, 상황 입력)은 절대 손대지 않는다.
5. **교사가 운전석** — AI는 도구. 4축 영점조절·페르소나 매칭의 최종 통제권은 사용자에게.

## 데이터 모델

### 핵심 엔티티
- **User** — 교사 본인 (Supabase Auth)
- **SchoolYear** — 학년도 단위 (1년 사이클)
- **Persona** — 학부모/동료/외부. `relationship_type`, 4축 디폴트, 관계 메모 보유
- **Conversation** — 임포트한 원본 텍스트 (비식별화된 형태로 저장)
- **Message** — 생성된 메시지. 생성 시 적용된 4축 값을 함께 저장
- **Reaction** — 학부모 답장 (사용자가 직접 복붙해 기록)

### 확장 슬롯 (1주차 스키마에는 포함, 활용은 후순위)
- `Persona.parent_persona_id` — 이전 학년 동일 학부모 연결
- `Persona.reference_persona_ids` — 작년 비슷한 페르소나를 참고로 연결
- `Message.category` — 메시지 분류 (사고/공지/상담 등)

## 4축 영점조절

| 축 | 0 | 10 |
|---|---|---|
| 격식 | 캐주얼 | 포멀 |
| 공감 | 사실 | 공감 |
| 단정 | 여지 | 단정 |
| 길이 | 짧게 | 길게 |

**단정 축은 법적 리스크와 직접 연결됨** — 단정값이 높을수록 확언/약속에 가까워지므로, AI 프롬프트에서 명시적으로 다룰 것.

### 도메인별 디폴트 (Persona 생성 시 초기값)
| relationship_type | 격식 | 공감 | 단정 | 길이 |
|---|---|---|---|---|
| 학부모 | 6 | 8 | 4 | 7 |
| 동료/상급자 | 7 | 5 | 6 | 5 |
| 외부 기관 | 9 | 3 | 5 | 6 |

### 작동 방식
- 페르소나에 4축 디폴트 저장
- 메시지 작성 화면 진입 시 디폴트 자동 적용
- 슬라이더 조작 → 디바운스 후 자동 재생성
- 매번 적용된 4축 값은 해당 Message 엔티티에 함께 저장 (재현·학습용)

## 비식별화 정책

### 자동 처리 (임포트 텍스트에만 적용)
- 학생 본명 → `[학생A]`, `[학생B]`...
- 학부모 본명 → `[학부모A]`, `[학부모B]`...
- 학교명·병원명·기관명 → `[학교]`, `[병원]`, `[기관]`
- 전화번호·주소 → `[전화번호]`, `[지역]`
- 학년·반 → `[반]`

### 손대지 않음 (사용자 직접 입력)
- 페르소나 이름란 — "민준이 어머니" 같은 호칭은 본명이 아니므로 그대로 둔다
- 관계 메모, 상황 입력 — 사용자가 의도해서 쓴 값

## 임포트 방식

- 사용자가 텍스트박스에 어디서든(하이톡·카톡·문자) **통째로 복붙**
- AI 단일 호출로 **분석 + 비식별화 동시 처리**
- 채널별 파서·정규식 파싱 만들지 말 것 — 단순함 우선

## 1주차 스코프 (4개 화면)

1. **로그인** — Supabase Auth, 이메일+비밀번호
2. **홈** — 페르소나 그리드, `relationship_type` 뱃지로 구분
3. **페르소나 만들기** — 4단계
   1. 관계 유형 선택
   2. 기본 정보 입력
   3. 대화 임포트 (복붙)
   4. AI 분석 결과 확인·수정
4. **메시지 작성** — 페르소나 선택 → 상황 입력 → AI 생성 → 4축 영점조절 → 복사

## 기술 스택

- **Next.js 14** (App Router)
- **TypeScript**
- **Tailwind CSS**
- **Supabase** (DB + Auth)
- **Anthropic API** (Claude) — 메시지 다음과 같이 사용:
  - 임포트 분석/비식별화: 단일 호출
  - 메시지 생성: 페르소나 컨텍스트 + 4축 + 상황 입력
- **Vercel** 배포

## 보안 정책

이 서비스는 **한국 초등학생·학부모 정보**를 다루므로 개인정보보호법·교육 현장 민감 사안 영역이다. AI 앱 특유의 공격 벡터(프롬프트 인젝션 등)도 함께 고려한다. 1주차 검증 단계라도 보안은 처음부터 보수적으로 간다. **이후 모든 코드 작성 시 아래 원칙을 준수하고, 위반을 발견하면 즉시 멈추고 사용자에게 알릴 것.**

### 일반 보안

#### 1. 시크릿 관리
- API 키 하드코딩 금지
- 모든 시크릿은 `.env.local` (git 제외)
- `.env.local.example`에는 키 이름만, 값은 비워둘 것
- Anthropic API 키는 **서버 사이드(Next.js API route)에서만** 호출
- 클라이언트에서 직접 Anthropic 호출 금지
- Supabase Anon Key만 클라이언트 노출 허용

#### 2. 데이터 격리
- 모든 Supabase 테이블에 **RLS(Row Level Security) 활성화**
- 사용자는 본인 데이터만 접근 (`user_id = auth.uid()`)
- "검증이니까 풀어두자" 절대 금지

#### 3. 비식별화
- 임포트 텍스트 비식별화는 **서버에서** 처리
- **원본 텍스트 저장 금지** — 비식별화 결과만 DB에 저장
- Anthropic API 요청에도 비식별화된 텍스트만 전송
- 로그·에러 추적에도 원본 금지

#### 4. 인증
- Supabase Auth 표준 패턴 사용
- 비밀번호 **8자 이상** 강제
- 세션 토큰은 **httpOnly 쿠키** (localStorage 금지)
- 비밀번호·토큰 `console.log` 금지

#### 5. CORS·HTTPS
- Vercel 자동 HTTPS 사용
- API route에 적절한 CORS 설정 — **와일드카드 금지**

#### 6. 에러 처리
- 사용자에게 DB 에러·스택 트레이스 노출 금지
- 일반화된 에러 메시지만 노출
- 상세 에러는 서버 로그에만 기록

#### 7. 깃 커밋
- `.env.local` 커밋 금지
- 실제 학생·학부모 정보 들어간 테스트 데이터 커밋 금지
- 커밋 전 `git diff`로 확인

### AI 앱 특화 보안

#### 8. 프롬프트 인젝션 방어
- 사용자 입력은 항상 **데이터로만** 전달 — 시스템 프롬프트와 명확히 분리
- `"ignore previous instructions"` 같은 패턴 감지 시 경고
- AI 응답이 시스템 프롬프트를 노출하는지 검증
- 출력에 다른 사용자 데이터가 섞이는지 검증

#### 9. AI 사용량 제한 (Rate Limiting)
- 사용자당 **일일 한도** (예: 100회)
- 사용자당 **월 한도** (예: 1000회)
- 한도 초과 시 정중히 차단
- 의심스러운 패턴(1분에 50회 등) 자동 차단

#### 10. 입력 검증·정제
- 텍스트 길이 제한
  - 페르소나 임포트: **50,000자**
  - 상황 입력: **2,000자**
- 비정상적 입력 패턴 거부
- 코드·SQL 인젝션 방어 — ORM 사용, raw SQL 금지

#### 11. AI 출력 검증
- AI 응답이 약속된 형식(JSON 등)을 따르는지 검증
- 응답에 **비식별화 안 된 본명·연락처 패턴**이 있는지 후처리 검사
- 검증 실패 시 사용자에게 노출하지 말고 서버 로그에만 기록

#### 12. Next.js 14 취약점 완화 조건 (1주차 한정)
- `next/image`의 외부 URL 사용 금지 (Image Optimizer `remotePatterns` DoS 회피)
- `next.config.mjs`에 `rewrites` 사용 금지 (HTTP request smuggling 회피)
- Server Actions(`"use server"`) 사용 금지 — 데이터 변경은 `/api` route로만 처리 (RSC deserialization DoS 회피)
- `next/image` 컴포넌트 사용 자체 금지 — 이미지는 `<img>` 태그 또는 Supabase Storage URL 직접 사용 (disk cache DoS 회피)
- 모든 인증 필요 페이지의 RSC 최상단에 인증 체크 배치 — 익명 요청이 RSC 렌더 트리에 진입하지 않도록 (Server Components DoS 회피)
- 위 조건 위반 시 즉시 Next 16 업그레이드 필요
- 1주차 종료 후 별도 브랜치 `chore/next16-upgrade`에서 Next 16으로 마이그레이션 (1순위 작업)
